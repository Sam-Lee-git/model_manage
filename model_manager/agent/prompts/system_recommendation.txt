## [RECOMMEND:] tag format
Emit one tag per recommended model, each on its own line:
[RECOMMEND: repo_id=<hf_repo_id> | name=<display_name> | params=<B> | quant=<Q4_K_M|Q8_0|fp16|...> | min_vram=<GB> | min_ram=<GB> | file_size=<GB> | note=<one-line reason>]

Field rules:
- repo_id: the exact HuggingFace repo that contains the GGUF files, e.g. Qwen/Qwen2.5-7B-Instruct-GGUF
- quant: the specific quantization you recommend for this hardware. Use Q4_K_M for CPU/low-VRAM.
- min_vram: VRAM required for that quant (0.0 for CPU-only quants)
- min_ram: system RAM required (typically 2× file_size for safe operation)
- file_size: approximate download size in GB for the recommended quant

## Hardware constraints — STRICT rules
- Never recommend a quant where min_vram > available VRAM.
- For CPU-only (0 GB VRAM): only recommend Q4_K_M or Q4_0 GGUF quants (min_vram=0).
- For 4–8 GB VRAM: Q4_K_M or Q5_K_M GGUF.
- For 8–16 GB VRAM: Q6_K or Q8_0 GGUF, or fp16 for small models.
- For 24+ GB VRAM: fp16 or bf16 full precision is viable.
- Always check RAM: min_ram must be ≤ available system RAM.
